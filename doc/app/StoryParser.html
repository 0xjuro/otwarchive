StoryParser

Parse stories from other websites and uploaded files, looking for metadata to harvest and put into the archive.

Constants

META_PATTERNS
CHAPTER_ATTRIBUTES_ONLY

These attributes need to be moved from the work to the chapter format: {:work_attribute_name => :chapter_attribute_name} (can be the same)

CHAPTER_ATTRIBUTES_ALSO

These attributes need to be copied from the work to the chapter

KNOWN_STORY_PARSERS

places for which we have a custom parse_story_from_[source] method for getting information out of the downloaded text

KNOWN_AUTHOR_PARSERS

places for which we have a custom parse_author_from_[source] method which returns an external_author object including an email address

KNOWN_STORY_LOCATIONS

places for which we have a download_story_from_[source] used to customize the downloading process

CHAPTERED_STORY_LOCATIONS

places for which we have a download_chaptered_from to get a set of chapters all together

SOURCE_LJ

regular expressions to match against the URLS

 972:       story = ""
 973:       Timeout::timeout(STORY_DOWNLOAD_TIMEOUT) {
 974:         begin
 975:           # we do a little cleanup here in case the user hasn't included the 'http://' 
 976:           # or if they've used capital letters or an underscore in the hostname
 977:           uri = URI.parse(location)
 978:           uri = URI.parse('http://' + location) if uri.class.name == "URI::Generic"
 979:           uri.host.downcase!

 981:           response = Net::HTTP.get_response(uri)
 982:           case response
 983:           when Net::HTTPSuccess
 984:             story = response.body
 985:           when Net::HTTPRedirection
 986:             if limit > 0
 987:               story = download_with_timeout(response['location'], limit - 1) 
 988:             else
 989:               nil
 990:             end
 991:           else
 992:            nil
 993:           end
 994:         rescue Errno::ECONNREFUSED
 995:           nil
 996:         rescue SocketError
 997:           nil
 998:         rescue EOFError
 999:           nil
1000:         end
1001:       }
1002:       if story.blank?
1003:         raise Error, "We couldn't download anything from #{location}. Please make sure that the URL is correct and complete, and try again."

This is the heavy lifter, invoked by all the story and chapter parsers. It takes a single string containing the raw contents of a story, parses it with Nokogiri into the @doc object, and then and calls a subparser.

If the story source can be identified as one of the sources we know how to parse in some custom/ special way, parse_common calls the customized parse_story_from_[source] method. Otherwise, it falls back to parse_story_from_unknown.

This produces a hash equivalent to the params hash that is normally created by the standard work upload form.

parse_common then calls sanitize_params (which would also be called on the standard work upload form results) and returns the final sanitized hash.
